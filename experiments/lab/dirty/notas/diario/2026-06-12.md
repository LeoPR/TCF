## Faxina pós-incidente OneDrive — artefato remoto DESKTOP [probatório]

Fechamento do loop do ADR-0021. Verificação direta no GitHub
(`git ls-remote --heads origin`): o remoto tinha **apenas** `main` —
o branch `main-DESKTOP-SG30VJF` **já não existia lá** (removido antes). O
que restava era só uma **ref de tracking local obsoleta**
(`refs/remotes/origin/main-DESKTOP-SG30VJF`) apontando pro snapshot
`c8a4673` (2026-05-26, "T-CODE-LAYERED-PIPELINE Fase 1 WELDED").

Conferência de datas/contenção antes de podar (lição "conferir DATAS antes
de apagar"): `c8a4673` é ancestral de `origin/main` e `git log
origin/main..c8a4673` é **vazio** — zero commits únicos, nada a perder. SHA
preservado aqui pra rastreabilidade; recuperável do próprio histórico de
main de qualquer forma. Podado via `git remote prune origin`. Repo `0/0`
com o remoto. Nenhuma cópia `*-DESKTOP-*` no working tree (já limpas antes).
Coerente com a leitura sóbria do incidente: latência local de sync, raro,
sem ação sistêmica.
