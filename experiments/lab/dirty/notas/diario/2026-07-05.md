## Direção 3 — cadeia Cython (revisão fraqueza #2: "o acelerador NUNCA chega ao usuário")

O 0.7.1 do PyPI é pure-Python **em silêncio**. Diagnóstico confirmado por repro e corrigido em 3 elos
(nada em `src/tcf`; só packaging/build/CI):

1. **`setuptools` faltava em `build-system.requires`.** O `hatch_build.py` roda `from setuptools
   import setup` num subprocess do build env **isolado** (PEP 517), onde só entra o que está em
   `requires`. Sem setuptools → subprocess exit 1 → `except` best-effort engole → wheel `py3-none-any`.
   O `[dependency-groups].dev` (que tem setuptools) **não** ajuda: é PEP 735, dev-local, não entra no
   env isolado.
   - **Prova**: build isolado com requires antigo → `tcf_format-0.7.1-py3-none-any.whl` (84KB, hook
     logou `CalledProcessError`); com `setuptools>=64` → `cp313-win_amd64.whl` (124KB) contendo
     `tcf/_core/detect.cp313-win_amd64.pyd`; instalada → `accelerated=True`, `impl=tcf._core.detect`.
2. **Falha silenciosa → sinalizável.** `hatch_build.py` honra `TCF_REQUIRE_ACCEL=1` (opt-in): se não
   compilar, build **falha alto** (RuntimeError) em vez de degradar. Default = best-effort (ADR-0020).
   - **Prova**: sem setuptools + `TCF_REQUIRE_ACCEL=1` → `pip wheel` exit 1, sem wheel, mensagem
     "A wheel seria pure-Python". Branch unit-testado (unset/0/false → warning; 1/true/on/yes → raise).
     Env **propaga** pro subprocess isolado (confirmado).
3. **Byte-equivalência VERIFICADA por teste (T-CI-3 fechado).** Job `accel` no `ci.yml` (matrix
   3.11/3.13, ubuntu): compila com `TCF_REQUIRE_ACCEL=1`, instala non-editable, confirma
   `accelerated=True` (falha em vez de skipar), roda `test_pyx_byte_equivalence` + regressão +
   real-world com `accel=True`. Local: **42 byte-equiv + D1-D9=1523B + RW=89616B** no caminho
   compilado. ADR-0020 ganhou NOTA (§3-bis): o "Gate .pyx==.py" passa de convenção a máquina.

**Distribuição (fica pro release 0.8.0, direção 2)**: cibuildwheel (wheels multi-plataforma no PyPI)
pra o usuário `pip install` pegar a extensão sem compilador. Plano já em
`estrategia-distribuicao-cython.md` (uv frontend, free-threading 3.14t, abi3). O fix de hoje garante
que o build **local/CI** compila; o cibuildwheel garante que o **PyPI** entrega compilado.

  1d (<300B) empata (confirma "<1KB não ajuda").

## Nested-TCF study + matriz de transmissão completada (owner)

Owner pediu, antes de fechar a matriz: estudar como o TCF trata o envio padrão (JSON request/response,
às vezes multi-camada/instrução aninhada) e se dá pra um "TCF aninhado similar ao JSON". Feito como
lab [2026-07-05-nested-tcf-study](../2026-07-05-nested-tcf-study/result.md) (FORK, não toca src/tcf).

- **Dois sentidos de "aninhado"**: o TCF já faz nesting de VALOR/afixo (HCC, herança M0 patricia-aninhado);
  o pedido é nesting de DOCUMENTO (árvore obj/array) — que o TCF não tem (API tabular). Distinção registrada.
- **Tese medida**: JSON aninhado = esqueleto escalar (config, pequeno) + arrays-de-objetos (tabulares,
  o volume). O array-de-objetos já é a unidade nativa do TCF. "TCF aninhado" = envelope JSON fino +
  1 bloco TCF por array (design B `nested_tcf`). RT 24/24.
- **Medido** (brotli-q11): response 744pts nested-TCF **990B = 71.9% do JSON-colunar** (−28%, herda o
  T1 cadenced); request batch n=500 nested-TCF **283B = 27% do raw** (array cadenciado+constantes →
  quase-constante em N). `flatten` ingênuo **explode** (path indexado). Config escalar pequeno: TCF 169B
  **perde** pra raw-JSON 140B (<300B, envelope = overhead). Feasibility **confirmada-conceitual**; é
  adaptador externo (gadget), não mudança de src/tcf.
- **Trace OBAT/HCC** por forma-tx salvo em `trace_output.txt` (o "experimento de sempre" que o owner
  pediu): forecast-block (cadência `*24|\2026`…), series-block (`ASSET_SYN_\00*\0*\1` + `*20|const`),
  adult (dict-coded `@`), pessoas (LIMITE: `!cpf` raw).

**Matriz completada** → [datasets/coverage-matrix.md](../../../../datasets/coverage-matrix.md) (corpo do
ticket T-DATA-TRANSMISSION-GROUPING, status `matrix-done`). Eixo forma-tx **refinado de 4 → 7 formas**:
upload-small (✗) · upload-batch (✓ escala) · download-bulk (✓) · download-cadenced (✓✓) ·
download-narrow-high-card (✗/LIMITE) · lazy-query (✓) · nested-response (✓ se conteúdo bulk/cadenced).
**Q1 = PARCIAL** (matriz consistente; vazios em XL/cadenced-real/µ/nested-real); **Q2 = PARCIAL 2/4**
