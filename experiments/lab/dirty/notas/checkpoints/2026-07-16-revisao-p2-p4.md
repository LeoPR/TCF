2. O commit `268608d` tornou `_dec_scalar` fail-loud para bool inválido, number não-numérico,

## Parecer técnico sobre P4 — não é decisão do owner

### P4a — array-em-array

Usar **count recursivo** como primeira hipótese. A representação carrega por nível a mesma informação
posicional que um repetition-level flat, reutiliza o primitivo `#count` e mantém a topologia
inspecionável. Não há evidência atual de que um stream flat adicional pague seu custo.

Gate de estudo:

- `[[1,2],[3]]`, matrizes, profundidade >2 e arrays internos vazios;
- arrays de arrays de objetos;
- null entre arrays e composição com P3b, com classificação explícita;
- counts internos truncados/excedentes, closers e tags inválidos;
- medida de custo por profundidade;
- round-trip exato + non-regressão flat + real-world snapshots antes de weld.

### P4b — raiz generalizada

Tratar em ato separado porque muda entrada/saída públicas. JSON possui **uma raiz**; “N-raízes” fica
somente como termo histórico. Array no topo preserva ordem. Uma raiz sintética só é aceitável com
um envelope/discriminador inequívoco que o decoder remova e que preserve o tipo original.

Gate de contrato:

- distinguir `[]`, `[{}]`, `{}` e o DatasetH `list[dict]` vigente;
- objeto único, array de escalares, escalar, string vazia e null na raiz;
- decode nunca expõe o envelope sintético;
- ordem de qualquer array preservada exatamente.
